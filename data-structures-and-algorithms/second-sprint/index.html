O que é uma Sessão de Recarga

Uma sessão representa o ciclo completo de recarga de um veículo elétrico em uma estação. Ela registra potência alocada, energia entregue, custo e duração.

EstaçãoA, B, C ou D
VeículoID + capacidade da bateria (kWh)
PlanoBásico 3,7 kW / Plus 7,4 kW / Ultra 22 kW
Statuscharging → done (ou queued)
Estrutura DSAMap<stationId, SessionObject>
🔋
Controle de Demanda e Throttling

A grade suporta no máximo 50 kW. Quando a soma das potências ultrapassa esse limite, todas as sessões são proporcionalmente reduzidas (throttling).

4 × Ultra (22 kW) = 88 kW solicitados Excede
Fator: 50 / 88 = 0,568 → cada estação recebe 12,5 kW Throttled
Estrutura DSAQueue (FIFO) para fila de espera
AlgoritmoDistribuição proporcional
↑ Potência nominalSem redução — demanda dentro do limite
↓ Potência reduzidaThrottling ativo — limite de 50 kW excedido

Protocolos OCPP 1.6 e MODBUS
OCPP 1.6 — mensagens JSON em array

Cada mensagem é um frame [tipo, id, ação, payload]. Tipo 2 = Call (envio), Tipo 3 = CallResult (resposta).

→ [2, "msg_001", "StartTransaction",
  {"connectorId":"A","idTag":"usr_1","meterStart":0}]
← [3, "msg_001",
  {"transactionId":4521,"idTagInfo":{"status":"Accepted"}}]
Estrutura DSAQueue (fila de frames pendentes)
MODBUS — Holding Registers

Registros de 16 bits endereçados por código. O valor bruto é multiplicado pelo fator de escala para obter a grandeza física.
